# 캘린더 대시보드 시스템 재가동 및 이관 가이드 (setup.md)

이 문서는 현재 컴퓨터에서 운영 중인 **세종시 지원정책/입찰정보 캘린더** 시스템을 다른 컴퓨터로 복사하거나, 시스템이 멈췄을 때 다시 가동하기 위한 절차를 설명합니다.

---

## 1. 사전 준비 (Prerequisites)

이관할 새 컴퓨터에 다음 프로그램들이 설치되어 있어야 합니다:

1.  **Python 3.10 이상**: 
    *   [python.org](https://www.python.org/)에서 설치 가능.
    *   설치 시 "Add Python to PATH" 체크 필수.
2.  **Git**: 
    *   데이터를 GitHub Pages에 자동 배포하기 위해 필요합니다.
    *   [git-scm.com](https://git-scm.com/)에서 설치.
3.  **GitHub 계정 및 권한**: 
    *   `deploy.bat`을 통한 자동 업로드를 위해 해당 저장소에 대한 Push 권한이 있어야 합니다.

---

## 2. 설치 및 환경 설정 (Setup Steps)

### 1) 프로젝트 복사
전체 폴더(`aCalendar`)를 새 컴퓨터의 원하는 위치(예: `C:\Project\aCalendar`)로 복사합니다.

### 2) 파이썬 라이브러리 설치
터미널(PowerShell 또는 CMD)을 열고 해당 폴더로 이동한 뒤 다음 명령어를 실행합니다:
```bash
pip install python-dotenv
```
*(참고: 이 프로젝트는 외부 라이브러리 의존성을 최소화하여 `python-dotenv` 외에는 기본 라이브러리만 사용합니다.)*

### 3) 환경 변수 설정 (`.env`)
폴더 루트에 있는 `.env` 파일을 메모장으로 열어 다음 항목을 본인의 정보로 업데이트합니다:
- `DATA_GO_KR_API_KEY`: 공공데이터포털(data.go.kr)에서 발급받은 '조달청 나라장터' API 키.
- `GMAIL_USER`: 업데이트 보고를 받을 Gmail 주소.
- `GMAIL_APP_PASSWORD`: Gmail의 '앱 비밀번호' (일반 비밀번호 아님).

### 4) 자동 경로 설정 확인 (`schedule_task.bat`)
현재 시스템은 프로젝트가 어느 폴더에 있든 자동으로 경로를 인식하도록 설정되어 있습니다 (`%~dp0` 가변 변수 사용). 
따라서 별도의 파일 수정 없이 바로 사용 가능합니다. 하지만 작동하지 않을 경우 메모장으로 `schedule_task.bat`을 열어 `cd /d` 뒤의 경로를 수동으로 입력해 보세요.

---

## 3. 초기 실행 확인 (Initial Run)

설정이 완료되었다면, 시스템이 정상 작동하는지 수동으로 실행해 봅니다.

1.  `execution\update_calendar_data.py`를 실행하여 데이터 수집 및 `event_data.js` 생성이 정상인지 확인합니다.
2.  `index.html`을 브라우저로 열어 캘린더에 일정이 정상적으로 표시되는지 확인합니다.
3.  `deploy.bat`을 실행하여 GitHub 저장소에 변경사항이 정상적으로 Push되는지 확인합니다.

---

## 4. 자동 업데이트 설정 (Windows 작업 스케줄러)

매주 월, 수, 금요일에 자동으로 데이터를 갱신하도록 설정합니다.

1.  Windows 검색창에 **'작업 스케줄러'** 입력 및 실행.
2.  우측 메뉴에서 **'기본 작업 만들기'** 클릭.
3.  **이름**: `Sejong_Calendar_Update` (자유롭게 입력).
4.  **트리거**: '매주' 선택 -> 월, 수, 금 체크 (시간은 오전 9시 권장).
5.  **동작**: '프로그램 시작' 선택.
6.  **프로그램/스크립트**: 프로젝트 폴더 내의 `schedule_task.bat` 파일 선택.
7.  **마침** 클릭 후, 목록에서 해당 작업을 찾아 '실행'을 눌러 테스트해 봅니다.

---

## 5. 문제 해결 (Troubleshooting)

- **API 호출 실패**: `.env` 파일의 `DATA_GO_KR_API_KEY`가 '디코딩' 버전인지 '인코딩' 버전인지 확인하여 교체해 보세요.
- **Git Push 오류**: 터미널에서 `git push`를 수동으로 입력하여 인증 오류가 발생하는지 확인하세요.
- **파일 경로 문제**: 모든 `.bat` 파일과 파이썬 스크립트 내부에서 사용되는 `C:\Users\...` 등의 기존 경로가 남아있는지 확인하고 현재 경로로 수정하세요.

---

> [!TIP]
> **3계층 구조 유의사항**
> - `directives/`: 시스템 운영 정책 및 SOP가 들어있습니다.
> - `execution/`: 실제 데이터 수집 및 업데이트를 수행하는 파이썬 스크립트가 있습니다.
> - `event_data.js`: 수집된 최신 데이터가 저장되는 파일입니다.
